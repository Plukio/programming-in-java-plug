Java has two main components ...

Java Runtime Environment (JRE)


The JRE provides the libraries, the Java Virtual Machine (JVM), and other components needed to run applications written in Java. It does not include developer tools such as compilers and debuggers. As a user, you would use the JRE to run Java programs on your system.


Java Development Kit (JDK)


The JDK includes the JRE as well as a set of development tools for writing and running Java programs. These tools include the Java compiler (javac), an archiver (jar), a documentation generator (javadoc), and other tools needed in Java development. As a programmer, you would use the JDK to develop Java applications.

Use right tools for the job

“If the only tool you have is a hammer, you tend to see every problem as a nail.” (Abraham Maslow)

C/C++

C/C++ family of languages are high-performance languages that give programmers a high level of control over system resources.

  • They are best applied where efficiency and performance or fine level of control is required.
  • More complicated syntax and lack of modern language conveniences makes them more difficult to master.
  • Developers need handle memory management manually which can introduce error, vulnerabilities and security risks.
  • High level complicated business requirements may take a lot of skill and of lines to implement.

Python

Python is a powerful and flexible, well suited for working with data or scripting.

  • Python has simple and readable syntax, which is often easy to learn for beginners.
  • Python is interpreted language, which makes it bit slower in certain situations than Java. Scientific computation libraries like NumPy and SciPy that use C in the backend for performance.
  • Python is popular amongst data science community because of its extensive libraries.
  • Python's flexibility may come at an expense of security and type-safety related runtime problems.

JavaScript / TypeScript

JavaScript / TypeScript is great for front-end web development allowing for interactive web application. Server-side JavaScript / TypeScript is also popular, running on Node.js

  • Easy to adopt, quick to develop in, flexible.
  • Big community, lots of frameworks and tools.
  • Relatively inefficient due to the fact that it is interpreted language and by design runs in a single thread.
  • Vanilla JavaScript is not type safe, not as robust and may be generally less safe.
  • Not the best choice for high-availability, critical systems.

Java

Java is a good combination of efficiency, power flexibility and safety

  • Not quite as efficient as C/C++, but good enough
  • Not quite as flexible s Python/JavaScript/TypeScript, but flexible enough
  • Bigger memory footprint than Python, but better safety
  • Java is mostly used for:
    • Web Applications - mostly the application backend
    • Mobile Applications - Android, although it is being replaced by Kotlin
    • Enterprise Applications - Java EE, banking and finance systems, healthcare systems, etc.
    • Cloud-based Applications - serverless applications, cloud functions
    • Scientific Applications & Big Data - AI and ML applications, Apache Spark, etc.
    • Gaming - mostly on Android (What popular game is in Java?)
    • IoT & Embedded Systems - IoT, embedded systems, smart devices
    • Middleware - such as web servers, WebSphere, Tomcat, ...
    • Desktop GUI Applications - JavFX, Swing

C#

One of Java's biggest competitors.

  • They are pretty similar in terms of usage and capabilities
  • While Java's design intent was to "Write Once, Run Anywhere" on JVM, C# is primarily used within the .NET framework and was originally designed for Windows development.
  • Today it can be run on different platforms using .NET Core
  • Both Java and C# are based on C/C++ syntax, however, C# has more syntactic sugar, providing features like properties, indexers and events directly in the language
  • Both have a lot of libraries and wide user base

Scala

Scala is very powerful language. Compiled into Java bytecode, it actually runs on JVM, just like Java.

  • Strongly type, very powerful, very safe, but more difficult to master
  • Modern syntax supporting type inference etc ...
  • High performance, provides language-level support for parallel processing and concurrency (actor model)
  • Functional by design, it supports both OOP and functional programing paradigm

Kotlin

Kotlin is maybe the current best substitute for Java. Compiles to Java bytecode, therefore runs on JVM as well.

  • Kotlin has all the features Java has and more
  • Modern language with modern language features such as explicit null safety, implicit type inference, and features supporting functional programming
  • Not quite as powerful as Scala, but much easier to learn
  • Supports concurrency through coroutines
  • Just like Scala, Kotlin is interoperable with Java

Object-oriented programming

Object-oriented programming

There are four main principles in OOP ...

  1. Encapsulation
    restricts access to the internal state of an object, protecting it from unauthorized access and ensuring data integrity.
  2. Inheritance
    enables class to have the same behavior as another class by inheriting it's properties and methods.
  3. Polymorphism
    allows us to define one interface or method that can have multiple implementations. It means that the same method or property could exhibit different behavior in different instances of object implementing given interface.
  4. Abstraction
    is a mechanism to represent the object features without exposing the actual implementation details. In other words, user of such object only needs to know what it does, not how it does it.

The two's complement

In Java, two's complement is used for the representation of signed integers. A negative number is represented as the two's complement of its absolute value. This allows positive and negative numbers to be added together directly.

  • For positive integers, the two's complement representation is the same as their binary representation.
  • For negative integers, you use binary representation of the absolute value of the number, then you invert each bit, and finally add 1 to the least significant bit (LSB).
Example: Binary representation of -5
  1. The binary representation of absolute value of -5 (=5) is 0101
  2. Inverting each bit gives you 1010
  3. Adding 1 to the least significant bit gives you the binary representation of -5, which is: 1011

Type Casting

Typecasting is a method of converting one data type to another. Keep in ming, that type casting may result in data loss!

Implicit Casting
This is also known as automatic type conversion. This is done by the JVM when we assign a value of smaller data type to a variable of a bigger data type. 

							
								int number = 100;
								long bigNumber = number;          // Implicit type casting, no data loss
								float floatNumber = bigNumber;    // Implicit type casting, no data loss

								System.out.println(floatNumber);  // 100.0

Explicit Casting
Also known as manual type conversion, we perform casting explicitly.
Upcasting = assigning smaller data type to larger data type - generally doesn't result in data loss.
Downcasting = assigning larger data type to smaller data type - may result in data loss. 

							
								double preciseNumber = 100.04;
								int wholeNumber = (int) preciseNumber;  // Explicit type casting, resulting in data loss (precision)

								System.out.println(wholeNumber);        // 100

null

In Java, null is a special value that represents the absence of an instance. It is used to indicate that a reference variable doesn't point to any memory location or object.

  1. Reference Variables:
    In Java, you can assign null to any reference variable (non-primitive types: objects, array, interface, etc).

  2. Default Value:
    For object reference types, the default value is null when they are defined as class members and not explicitly initialized.

  3. NullPointerException:
    If you try to invoke a method or access a property on a reference variable with a null value, you will get a NullPointerException. This is a runtime exception in Java.

  4. Comparison:
    You can use null in comparison operations. For example, to check if an object is null, you can use if (object == null).

  5. Assigning Null:
    Assigning null to a variable makes it eligible for garbage collection if there are no other references to the object.

  6. Method Argument:
    null can be passed as an argument to a method and can also be returned from a method.

Bitwise Operators

& binary and sets bit to 1 if both bits at the same positions are 1 
									
										int a = 5; // binary: 0101
										int b = 3; // binary: 0011

										int result = a & b; // binary: 0001, decimal: 1
| binary or sets bit to 1 if either of bits at the same positions are 1 
									
										int a = 5; // binary: 0101
										int b = 3; // binary: 0011

										int result = a | b; // binary: 0111, decimal: 7
| binary xor sets bit to 1 if of bits at the same positions are different 
									
										int a = 5; // binary: 0101
										int b = 3; // binary: 0011

										int result = a ^ b; // binary: 0110, decimal: 6
~ binary complement operator flips all the bits of the operand; 1s become 0s, and 0s become 1s 
									
										int a = 5; // binary: 0101

										int result = ~a; // binary: 1010, decimal: -6
<< signed left shift moves all bits by a given number of bits to the left 
									
										int a = 5; // binary: 0101

										int result = a << 1; // binary: 1010, decimal: 10
>> signed right shift moves all bits by a given number of bits to the right 
									
										int a = 5; // binary: 0101

										int result = a >> 2; // binary: 0001, decimal: 1
>>> unsigned right shift same as the signed right shift, but the vacant leftmost position is filled with 0 instead of the sign bit 
								
									int a = -5; // binary equivalent of -5: 11111111111111111111111111111011

									int result = a >>> 1; // shifts right with zero fill

Ternary Operator

Java supports tenary operator as a shorthand for if else, which can come in handy when conditionally assigning values.
It is less code, but it also helps us avoid variable mutation, thus limiting the chance of error. 

variable = (condition) ? expressionTrue :  expressionFalse;

Consider this code: 
					
						long temperature = 33;
						String clothes = null;

						if (temperature > 20) {
							clothes = "T-Shirt";
						} else {
							clothes = "Sweater";
						}
You can write the same thing in shortened form 
					
						if (temperature > 20) clothes = "T-Shirt"; clothes = "Sweater";
Or it can be written using tenary operator like so: 
					
						String clothes = temperature > 20 ? "T-Shirt" : "Sweater";

For Loop

For loop executes a block for a given number of iterations. The evaluation of for loop is devided into 3 statements:

  • init statement is executed once before the execution of the code block
  • condition statement defines the condition for executing the code block
  • after statement is executed every time after the code block has been executed
						
							for (init; condition; after) {
							  // code to be executed
							}

This would be an example of executing code in a loop 10 times, where the loop is controlled by variable i with initial value of 0, incremented by 1 each time the loop is executed until the condition of i < 10 is no longer true. 

						
							for (int i = 0; i < 10; i++) {
							  // code to be executed
							}

You can also nest loops.

						
							for (int i = 0; i < 10; i++) {
								for (int j = 0; j < 3; j++) {
									// code to be executed
								}
							}

For Each Loop

If you have an array, can iterate it using for loop.

						
							String[] languages = {"Java", "Python", "C++", "JavaScript"};

							for (int i = 0; i < languages.length; i++) {

								System.out.println(languages[i]);

							}

You can also use more convenient for each loop.

						
							String[] languages = {"Java", "Python", "C++", "JavaScript"};

							for (String language : languages) {

								System.out.println(language);

							}

Break / Continue

The break statement is used to end the loop immediately.

						
							for (int i = 0; i < x; i++) {

								if (breakConditon) {
									break;
								}

							}
							// if breakConditon was met, the loop will end

The continue statement is end current iteration, but the loop will continue with next iteration.

						
							for (int i = 0; i < x; i++) {

								if (continueCondition) {
									continue;
								}

								// if continueCondition was met, this code will not be executed
							}
